<Heading level={2} number="2">What Is a Vibe Coder?</Heading>

A Vibe Coder relies heavily on AI to generate code, often without fully understanding what is being produced. The term comes from the feeling of "going with the flow" — prompting, accepting, and shipping without deep review.

<Spacer />

<Heading level={3} number="2.1">Typical Workflow</Heading>

  language="javascript"

function TodoApp() {
  // Generated code — barely reviewed
  return (
    <div>
      <h1>My Todos</h1>
      {/* AI wrote this. Looks fine. Ship it. */}
    </div>
  );
}`}
/>

<Spacer />

<Heading level={3} number="2.2">Common Traits</Heading>

<Paragraph>Vibe coders <Text bold>move fast</Text> but often struggle to debug issues they cannot describe. They treat AI as an oracle rather than a pair programmer. The code works — until it does not, and then they are stuck.</Paragraph>

<Spacer />

<Heading level={2} number="3">What Is an AI-Assisted Developer?</Heading>

An AI-Assisted Developer uses AI as a <Text italic>tool</Text>, not a replacement for understanding. They review every suggestion, refactor when needed, and treat generated code as a starting point rather than a final product.

<Spacer />

<Heading level={3} number="3.1">Typical Workflow</Heading>

<CodeBlock
  language="javascript"
  filename="ai-assisted-developer.js"
  highlightLines={[4, 9, 14]}
  code={`// Prompt: "Generate a debounce hook"
// The AI-Assisted Developer reviews and refines

function useDebounce<T>(value: T, delay: number): T {
  const [debounced, setDebounced] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debounced;
}

// Reviewed: added generics, proper cleanup, correct deps`}
/>

<Spacer />

<Heading level={3} number="3.2">The Mindset</Heading>

The key difference is <Text bold>ownership</Text>. AI-Assisted Developers own every line of code in their project, whether they typed it or not. They can explain, modify, and debug any part of the system.

<Spacer />

<Heading level={2} number="4">Side-by-Side Comparison</Heading>

<CodeBlock
  language="javascript"
  filename="comparison.js"
  code={`// Vibe Coder approach
function calculateTotal(items) {
  // AI wrote this — not sure how it works
  return items.reduce((acc, item) => acc + item.price * item.qty, 0);
}

// AI-Assisted Developer approach
function calculateTotal(items: CartItem[]): number {
  // Reviewed: handles edge cases, typed properly
  if (!items?.length) return 0;
  return items.reduce((sum, item) => {
    if (item.price < 0 || item.qty < 0) {
      throw new Error("Invalid item data");
    }
    return sum + item.price * item.qty;
  }, 0);
}`}
/>

<Heading level={2} number="5">Which Path Is Right for You?</Heading>

<Heading level={3} number="5.1">Go Vibe Coder When</Heading>

You are prototyping, building a personal project, or exploring an unfamiliar technology. Speed matters more than correctness or maintainability.

<Spacer />

<Heading level={3} number="5.2">Go AI-Assisted When</Heading>

You are working on a <Link href="https://en.wikipedia.org/wiki/Production_environment">production system</Link>, collaborating with a team, or building something that needs to last. Quality, testability, and maintainability matter.

<Spacer />

<Paragraph lead>Think of it like driving:</Paragraph>

<Heading level={2} number="12">Final Thoughts</Heading>

<Paragraph lead>AI is not going anywhere. The question is not whether to use it — it is how.</Paragraph>

The <Text bold>Vibe Coder</Text> approach works for throwaway projects and rapid exploration. The <Text bold>AI-Assisted Developer</Text> approach scales to production systems, team collaboration, and long-term maintainability.

Neither is inherently wrong. The best developers learn to switch between modes depending on the context. Use AI to accelerate, not to abdicate.
